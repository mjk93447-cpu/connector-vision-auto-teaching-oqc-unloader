# 1차 목표 사양 — 커넥터 핀 자동 마스킹 (YOLO26)

별도 티칭 없이 핀 위치를 마스킹할 수 있는 오프라인 Windows EXE.

---

## 1. 아키텍처

- **모델**: Ultralytics **YOLO26** (객체 탐지). 핀 크기가 작아 소형 객체 탐지에 유리.
- **학습**: 1회 로컬 학습 후 추가 티칭 불필요

---

## 2. 학습 단계 (1회)

### 입력

| 항목 | 설명 |
|------|------|
| 마스킹 전 커넥터 탑뷰 사진 | 원본 이미지 × 10 |
| 마스킹 후 커넥터 탑뷰 사진 | 핀 위치 레이블(초록색) × 10 |
| 엑셀 파일 | 위/아래 핀 개수, OK/NG, 좌우간격 × 10 (세트) |

### 동작

- 위 데이터로 YOLO 모델 학습
- 학습된 모델 저장 (추후 추론에 사용)

### 마스킹 표시 (학습용 레이블)

- 핀 영역 전체를 **초록색 점**으로 표시

### 위/아래 핀 구분

- 위핀: 수평으로 가로 정렬된 한 줄
- 아래핀: 검은색 여백(핀 크기 약 5배) 아래, 수평으로 가로 정렬
- 여백이 커서 혼동 가능성 거의 없음

### 엑셀 양식

- 헤더 **고정**, 양식 항상 동일. 학습 시 10개 엑셀 세트 제공.

---

## 3. 추론 단계 (학습 이후)

### 입력

- 마스킹 없는 커넥터 탑뷰 사진

### 출력

| 항목 | 설명 |
|------|------|
| **자동 마스킹 이미지** | 입력 이미지에 위핀 20개·아래핀 20개 위치를 **초록색 점**으로 마스킹 |
| **핀 개수** | 위핀 개수, 아래핀 개수 |
| **OK/NG** | 각 20개 → OK, 그 외 → NG |
| **엑셀** | 학습 시 인식한 엑셀 형식을 그대로 사용: 핀 개수, OK/NG, 핀 사이 좌우간격(mm) |

### 좌우간격 계산

- 마스킹 위치 좌표 + 핀 크기(0.5mm) 기준으로 mm 단위 좌우간격 산출
- 학습 엑셀 양식과 동일하게 저장

---

## 4. 요약

| 단계 | 입력 | 출력 |
|------|------|------|
| **학습 (1회)** | 마스킹 전/후 사진, 엑셀 | 학습된 모델 |
| **추론** | 마스킹 없는 사진 | 마스킹 이미지, 엑셀(핀 개수, OK/NG, 좌우간격) |

---

## 5. 산출물

- **오프라인 Windows EXE**: 학습·추론 모두 로컬 실행

---

## 6. 성능 지표 (핀 감지)

| 지표 | 목표 | 설명 |
|------|------|------|
| **Recall** | 100% | 위 20개·아래 20개 중 **하나라도 놓치면 안 됨** |
| **Precision** | 100% | 20개보다 **더 감지해서도 안 됨** (과탐지 금지) |
| **영역 마스킹** | 정확 | 위치는 recall보다 상대적으로 덜 중요 |

학습 시 마스킹 전/후 이미지 비교를 통해 어노테이션 품질 검증. 소형 핀 감지에 맞게 YOLO26 imgsz, augmentation 조정.

---

## 7. 개발 가이드

- **질문**: 애매하거나 이해되지 않는 부분은 바로 질문.
- **설명**: 커넥터/핀 사진은 보안상 제공 불가. 필요 시 말로 구체적으로 묘사·설명 가능.
